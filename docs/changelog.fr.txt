# ST5 (Français)
Meilleure gestion des erreurs et encore toujours plus de corrections.


## Changelog
- Refonte des commentaires dans le code.
- Refonte du chargement des fichiers .css et .js, maintenant ça se passe directement dans le fichier config.php des thèmes, ça permet enfin aux thèmes de charger les fichiers qu'ils veulent sans toucher au code du header.php.
- La colonne "wsname" de la table config passe de maximum 30 caractères à 25.
- Le nom d'un article, d'une page et du site doivent faire au minimum 3 caractères.
- Amélioration des commentaires dans le code (encore).
- Amélioration du nommage des variables dans le code (encore).


## Ajouts
- Ajout de la gestion des erreurs HTTP 413 et 415.
- Ajout des messages d'erreurs en cas où si le nom de l'article ou de la page dépasse les 50 caractères ou fait moins de 3 caractères lors de la création ou édition.
- Ajout d'un message d'erreur en cas où si la description de l'article dépasse les 75 caractères lors de la création ou édition.
- Ajout de la possibilitée d'éditer le nom d'utilisateur, l'adresse mail, le grade et le mot de passe des utilisateurs à partir du panel.
- Ajout des messages d'erreurs en cas où si le nom du site dépasse les 25 caractères, si la description dépasse les 75 caractères ou si encore le chemin vers le thème / thème du panel est introuvable dans la page "configuration" dans le panel.


## Corrections
- Correction d'un bug qui faisait qu'il était possible d'utiliser un nom d'utilisateur non-autorisée (c'est-à-dire avec des caractères spéciaux) à partir des paramètres du compte.
- Correction d'un bug qui faisait qu'il était possible d'utiliser un mot de passe avec moins de 8 caractères à partir des paramètres du compte.
- Correction d'un bug qui faisait qu'il était impossible de modifier son mot de passe depuis quelques mises à jours.
- Correction d'un bug qui faisait que la page inscription essayait de se chargeait quand on essayait d'y accéder en étant connecté.
- Correction d'un bug qui faisait que si PamplemousseCMS était installé dans un dossier alors le .htaccess ne fonctionnait pas.
- Correction d'un bug qui faisait que les pages "Editer une catégorie" et "Editer une page" n'avaient pas eu la mise à jour de la refonte des alertes.
- Correction d'un bug qui faisait que la page "Editer une page" n'avait pas eu la mise à jour de suppression des lignes inutilisées.
- Correction d'un bug qui faisait que /homepage redirigeait vers une erreur 404 alors qu'elle était censée redirigé vers l'index.
- Correction d'un bug qui faisait qu'il était possible sur la page "configuration" du panel d'entrer comme configuration des chemins inexistants pour un thème / thème du panel.
